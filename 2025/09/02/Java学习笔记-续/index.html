多catch情况：

特别的：当chatch的异常为并列关系时，捕获第一个异常之后，try中代码停止运行，后续代码无法继续运行，此时可以将后续代码放入catch中，作为catch处理的一部分运行后续代码，继续捕获异常。

finally关键字：

程序中，不管有没有异常，最终部分都有部分代码必须执行，这部分代码被写在finally中。try仅仅只是为了配合finally而存在的，其中异常并没有被捕获（因为没有catch，如果在finally前面加上catch{}那就有捕获异常的过程，不论如何，finally中的代码一定会被执行）
try{ } finally{一定会被执行的代码}

特别的：当try，catch，finally中都有return时（return语句在一个方法中只会执行一个）
此时finally会优先执行return语句，但是在非return语句时会按顺序正常执行

集合的介绍

集合：可以存储对象，随着对象的增加，容量可以自动扩容。在JDK中形成集合体系（不是一个类或一个接口）。

JDK中提供的集合分为三大类：

List集合：存储可以重复的数据，有下标

Set集合：不能存储重复数据（使用较少）

Map集合：存储key-value这种有键值对关系的数据

Collection接口

Collection接口：称为单列集合的顶层接口。List接口与Set接口都是Collection的子接口。

Collection接口中定义集合的一些共性的操作方法。

Iterator接口（迭代器）

迭代器的细节：

  1. 每个迭代器对象获取到之后，会关联到当前的集合容器对象，这时这个迭代器对象只能去遍历这个集合对象
  2. 每次调用hasNext方法的时候，判断当前迭代器隐式指针（光标）后面有没有元素，若有元素返回true，这时建议只调用一次next方法，每调用一次next方法，指针会向后移动一个位置。
  3. 若集合已经遍历完成， 这时无法再次使用这个迭代器对集合从前往后遍历。
  4. 遍历集合的时候，不需要对集合进行添加、修改、删除操作，因为迭代器无法感知到集合中的元素在遍历过程中的变化，若需要删除，只能调用迭代器对象中提供的remove()方法

List接口：列表接口，它主要在针对Collection接口的方法之外提供可以根据下标操作的方法。

List接口中的方法

2.3、ArrayList类

ArrayList类：它是List接口的实现类，实现List接口中的所有方法，它的底层采用的可变数组作为底层存储结构，多线程操作不安全。遍历查询效率较高。

可变数组：数组是不可变，长度是固定，但ArrayList集合的底层，在给集合中添加元素的时候，若达到集合的上限，这时ArrayList内部会调用grow方法对集合进行扩容操作，在原来容量的基础上增加容量的0.5倍，得到一个新的数组，将原来数组中的数据通过Arrays中的copyOf方法复制到新数组中，并销毁原来的数组。

2.4、LinkedList类

LinkedList类：它也是List接口的实现类，底层采用双向链表结构，LinkedList集合提供除List接口之外的其他方法：主要围绕链表的特点，可以对集合中的元素进行头尾操作。增删快，查询慢。

可以根据头尾的操作，模拟数据结构:

队列结构：先进先出（FIFO），后进后出（LILO）

堆栈结构：先进后出（FILO），后进先出（LIFO）

头尾添加的方法：addFirst、addLast、offerFirst、offerLast、

头尾删除的方法：removeFirst、removeLast、peekFirst、peekLast、

头尾获取的方法：getFirst、getLast

2.5、Vector集合

Vector：它是JDK1.0时期的产物，底层采用的也是可变数组，在JDK1.2之后被ArrayList代替，Vector它底层是线程安全的，

3.2、HashSet类

HashSet集合：它的底层采用的哈希表结构，不保证元素的存取顺序。

在给Hash结构中存储数据的时候：

  1. 先根据被存储的数据，调用hashCode方法，这时会得到哈希值，通过哈希值计算当前需要存储的数据在哈希表中的位置
  2. 若根据数据特征调用hashCode方法之后，计算的位置上若没有数据，直接存储。
  3. 计算的位置上已经存在数据，这时称为哈希冲突，这时需要再去调用存储的数据对应的equals方法，判断当前正要存储的数据与已经在当前位置上的数据是否是同一个，这时equals若返回的true，表示是同一个，就不存储。若equals返回的是false，表名当前存储的数据与当前位置上的数据不是同一个对象，那么当前数据也要存储。

3.4、TreeSet类

TreeSet集合：它底层采用的红黑树结构（二叉树），存储的元素可以进行排序。

TreeSet集合底层会根据存储的对象所属的类实现的Comparable接口，进行对象的大小比较。通过比较完成存储的数据排序。

一般要求：给树结构中存储的对象，对象对应的类应给去实现Comparable接口，保证对象能够进行比大小。

3.5、Comparator接口

当给Tree结构中存储对象的时候，若对象所属的类已经实现Comparable接口，对象自身也具备compareTo方法进行比较。

若当前Tree结构的排序方式与compareTo中已经存在的比较方式不同，这时就不能使用compareTo方法。

可以在创建Tree结构集合时，传递Comparator对象，这时再给Tree结构中存储数据的时候，就不会使用数据自身的compareTo进行比较，而是采用传递的Comparator对象中的compare方法进行比较。

3.6、Comparable和Comparator区别

Comparable：它是让某个对象自身具备比较功能。被比较的对象所属的类必须去实现Comparable接口，复写ComparaTo方法

Comparator：它和比较的对象没有什么关系，当需要比较的时候，直接new比较器的对象或者实现类的对象，就可以对任意两个对象进行比较。

LinkedHashSet：底层是哈希表+链表，可以保证元素的唯一和头尾特性。

5、集合总结

Collection接口：单列集合，它们下面有两个重要的子接口

​ List接口：代表所有可以重复的元素的集合，而且有下标，多线程操作不安全。

​ ArrayList类：底层可变数组，有下标，查询快，增删慢

​ LinkedList类：底层链表，有头尾操作的方法，查询慢，增删快

​ Vector类：它是JDK1.0时期的集合，底层也是可变数组，什么操作都慢，线程安全

​ Set接口：代表的不重复的集合，多线程操作不安全。

​ TreeSet类：底层红黑树，可以对存储的对象进行排序（排序依赖Comparable接口，或给集合传递Comparator对象）

​ HashSet类：底层是哈希表结构，存储的对象所属的类需要复写hashCode和equals方法

​ LinkedHashSet类：底层链表+哈希表

Iterator接口：它是用来遍历Collection接口下的所有集合的。

Enumeration接口：它也是一个迭代器，用于遍历Vector等早期的集合，功能与Iterator相同

Map接口：双列集合，它下面有2个重要的实现类

​ HashMap类：底层哈希表，作用key上，若key是自定义类也需要复写写hashCode和equals方法

​ LinkedHashMap类：底层链表+哈希表，作用key上

​ TreeMap类：底层红黑树，作用key上，若key是自定义类要实现Comparable接口，或者给集合传递Comparator对象

​ Hashtable类：它是JDK1.0时期的集合，底层使用哈希表，作用key上，线程安全。

​ Properties类：它需要与IO结合，可以读写具有key=value的文件数据。

1、网络介绍

网络：两台设备之间通过有线、无限、蓝牙、星闪、卫星等连接在一起，可以进行数据交互。这时两台设备之间构建一个属于它们的网络。

网络模型：OSI模型，将网络划分成7层

IP：它处于网络中的某个设备的身份标识，在网络中需要找到通信的对方，必须先知道对方的IP地址。

端口：设备中最终通过某个应用软件在接受数据，这时由于设备中会运行多个软件，每个运行的软件在当前设备都都有唯一的一个数字标识，称为当前这个软件在这个设备中的端口号。端口号从065535之间，一般01024之间的端口号留给操作系统使用。

协议：通信双方需要遵守的通信规则，在OSI模型的传输层中TCP和UDP。

域名解析：分为两步：本地解析、DNS解析

本地解析：在通信设备上，当输入网址（域名）之后，这时设备会在本地找hosts文件，在其中查询当前的域名与ip地址有没有映射关系

hosts文件位置：C:\Windows\System32\drivers\etc\hosts

DNS解析：若本地解析没有成功，这时才会走DNS服务器进行解析，查找域名和IP地址映射关系。找到就继续拿着IP地址去访问具体设备，若还是没有找到，这时浏览器就会提示相关的错误信息

多线程

1、多线程介绍

手机、平板、电脑上安装的软件，最终都是安装系统磁盘上，但是当运行软件的时候，将软件对应的程序加载到设备的内存中。

一台电脑或手机同时运行不同的程序，这些程序在内存中如何被管理呢？

1.1、进程介绍

操作系统会针对每个正在运行中的程序进行有效管理，启动程序的时候，会在内存中划分出每个程序运行时各自空间。划分的这些空间针对每个运行的程序而言，就是它们所在内存中的进程空间。

进程：一个正在内存中运行的程序。

1.2、线程介绍

进程：是一个程序在内存中运行空间（管理正在进行程序）

每个运行的软件中都有很多的程序文件与对应的代码功能需要独立去运行。在程序中，或给每个需要独立运行的程序代码分配独立内存空间（包含在进程中），这时每个负责真正运行代码的空间，称为一个线程。

线程：才是真正干活，负责代码执行单元。

1.3、多线程介绍与执行

多线程：现在的手机、平板、电脑等，几乎都是多用户，多任务的操作系统。在操作系统同时运行多个软件，每个软件中又允许同时运行多个线程来共同完成某些软件执行所需要的任务。

例如：微信程序，微信启动之后，在内存中最少有一个进程（子进程），同时可以和不同的人聊天，这时每个聊天窗口，都是一个独立程序块，就可以看作为一个独立的进程。

当我们使用手机、平板、电脑等设备的时候，目前设备都具备多核心，多线程能力。允许同时多个任务和线程并行执行。

在执行过程中，CPU会在多个线程执行进行高速切换。

  • 继承Thread类，子类就是线程类，子类属于线程中的一种，若某个类已经有所属父类，这时是无法再继承Thread类。

3.2、线程安全问题

CPU在执行完任何一个指令（句柄）之后，都可以从当前线程切换到其他线程上去，这个是cpu执行代码的标准，不能改变。

多线程运行出现的问题（安全问题）：本质原因是CPU的高速切换导致数据不一致。

从代码角度去分析：若多个线程没有对共享的数据进行操作，也不会有安全问题发生。多个线程操作共享数据num才导致票号不一致的情况。

解决的方案：采用JDK中提供的同步技术。同步技术实现有两种实现方案：同步代码块、JDK5版本中提供的Lock接口

同步代码块上的锁对象，可以是任意对象，只要保证对象唯一即可。

注意事项：添加同步的时候，一定是添加在操作共享数据相关的代码上，只要与共享数据无关的代码， 不要加同步，否则会影响程序执行效率。

3.3、JDK5种的Lock接口

使用synchronized关键字完成的线程的同步操作，锁的获取和释放由JVM底层去完成，隐式的同步锁操作。

在JDK5中提供Lock接口，它将过去的隐式的同步锁的操作（获取和释放）改为由程序员手动编写实现锁的获取与释放。

  • lock( ) : 线程获取锁
  • unLock( ) : 线程释放锁

7.1、什么是连接池

数据库连接池：本身可以理解为一个容器，其中存储的是数据库的链接，当程序中需要使用数据库连接操作数据库的时候，不是直接问数据库获取连接，而是从数据库连接池容器获取一个，在使用完之后，还给连接池即可。

数据库连接池存储的连接，它是可以给不同的程序提供操作数据库，然后当连接使用完成之后还给连接池。
